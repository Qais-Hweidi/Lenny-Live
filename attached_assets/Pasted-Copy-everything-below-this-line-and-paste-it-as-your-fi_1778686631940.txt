## The product concept (the full vision)

**"Lenny Live"** is a debate room where a user asks a product/career question and **two or three AI personas — each grounded in actual transcript quotes from real podcast guests (Shreyas Doshi, Teresa Torres, Reid Hoffman, Mira Murati, etc.) — argue it out**. Lenny moderates with sharp follow-ups in his interviewing style. Every claim a persona makes shows a **source citation pill** (episode + timestamp) pulled from the real transcripts.

The user can **interrupt** the debate. The end-state is a **shareable summary card** with where the panel agreed, where they split, and the one quote to remember.

The final product is voice-first. **We are building it in phases.**

## Data pipeline (build-index.ts)

This runs **once** at project setup, output is committed to the repo so the server starts instantly.

1. Read every `.md` file in `/data/podcasts/` and `/data/newsletters/`
2. For each file, parse front-matter / first lines to extract: guest name, episode number, title (cross-reference with `index.json`)
3. Chunk each transcript into ~500-token segments with ~100-token overlap
4. Estimate a timestamp for each chunk (linear interpolation: chunk position / total chunks × episode duration from `index.json` if available; otherwise just chunk index)
5. Embed every chunk with `text-embedding-3-small`
6. Save to `/data/embeddings.json`: an array of `{ guest, episode, title, timestamp, text, embedding }`

## Panel-selection endpoint logic

`POST /api/panel { question }` →

1. Embed the question
2. Compute cosine similarity to every chunk
3. Group top 50 chunks by guest
4. For each guest, summarize their stance in one sentence (LLM call, grounded in their top chunks)
5. From the top 5-6 candidate guests, **pick 3 whose stances are most divergent** (LLM call: "given these stances, pick the 3 that disagree most strongly")
6. Return `{ guests: [{ name, stance, episode, title, timestamp, color }, ...] }`

## Debate endpoint logic

`POST /api/debate { question, guests, interjection? }` → returns Server-Sent Events stream

The system prompt to GPT-4o-mini:

```
You are orchestrating a live debate between {guest1}, {guest2}, and {guest3}
about: "{question}".

Lenny Rachitsky moderates in his real interviewing style (calm, probing,
asks "but why" follow-ups, surfaces tensions).

Each panelist has these grounding quotes from their real podcast appearances:
{guest1.relevant_chunks}
{guest2.relevant_chunks}
{guest3.relevant_chunks}

Produce a 6-8 turn debate. Each turn is JSON:
{
  "speaker": "Shreyas Doshi" | "Teresa Torres" | "Reid Hoffman" | "Lenny",
  "text": "...",
  "source": { "episode": 142, "title": "...", "timestamp": "23:14" } | null
}

Rules:
- Panelists must disagree at least twice
- Lenny asks a probing follow-up between turns 3-4 and after turn 6
- When a panelist makes a strong claim, ground it in a real quote from
  their chunks and set `source` to that episode
- Keep each turn under 60 words
- Stream as a JSON array, one object at a time
```

Stream the response back to the client and render each turn as it arrives.
